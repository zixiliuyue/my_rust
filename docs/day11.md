# Day 11：线程、channel 和并发边界

## 先运行

```bash
cargo run --bin day11_concurrency
```

## 再读代码

重点阅读 `src/bin/day11_concurrency.rs`：

- `thread::spawn` 创建线程。
- `move` 把任务数据移动到线程里。
- `mpsc::channel` 用消息传递汇总结果。
- `drop(sender)` 关闭最后一个发送端，让接收端迭代结束。

## 底层原理

Rust 的并发安全来自所有权和类型系统。跨线程传递数据时，编译器会检查值是否能安全发送。消息传递把共享可变状态减少到最低，更适合新手先建立并发边界。

## 练习

- 增加一个失败结果字段，模拟某个 worker 失败。
- 把结果从字符串改成结构体。
- 思考什么时候应该用 `Arc<Mutex<T>>`，什么时候应该用 channel。

## 常见坑

- 不要让线程引用主线程里的临时变量。
- 多线程输出顺序通常不稳定，需要排序后再断言。
- channel 发送失败通常表示接收端已经关闭。
