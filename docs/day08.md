# Day 8：生命周期、切片和引用有效期

## 先运行

```bash
cargo run --bin day08_lifetimes
```

## 再读代码

重点阅读 `src/bin/day08_lifetimes.rs`：

- `first_word` 返回借用自入参的 `&str`。
- `longer<'a>` 显式声明两个引用和返回值之间的生命周期关系。
- `Note<'a>` 演示结构体里保存引用时为什么需要生命周期参数。

## 底层原理

生命周期不会延长任何值的存活时间，它只是让编译器验证引用不会悬垂。`&str` 切片保存的是地址和长度，通常不拥有字符串内容。

## 练习

- 写一个 `last_word(text: &str) -> &str`。
- 把 `Note` 增加 `source` 字段，继续使用同一个生命周期参数。
- 尝试返回函数内部创建的 `String` 的引用，观察编译器错误。

## 常见坑

- 生命周期标注描述关系，不负责分配内存。
- 大多数函数可以依赖生命周期省略规则，不需要手写 `'a`。
- 返回引用时一定要确认被引用的值由调用方拥有。
