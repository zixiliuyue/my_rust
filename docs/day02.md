# Day 2：所有权、移动和借用

## 先运行

```bash
cargo run --bin day02_ownership
```

## 再读代码

重点阅读 `src/bin/day02_ownership.rs`：

- `String::from(...)` 创建堆上可增长字符串。
- `let new_owner = owner;` 会移动所有权。
- `clone()` 会复制堆数据。
- `&String` 是不可变借用，`&mut String` 是可变借用。

## 底层原理

`String` 变量本身通常在栈上，里面保存指针、长度和容量；真正的文本字节在堆上。移动所有权时，Rust 只移动这组三元数据，不复制堆里的文本。

Rust 禁止移动后继续使用旧变量，是为了避免两个变量在离开作用域时释放同一块堆内存。借用则让函数临时访问值，但不接管释放责任。

## 练习

- 删除 `clone()`，观察什么时候会出现“borrow of moved value”。
- 写一个函数接收 `&String` 并打印首个字符。
- 写一个函数接收 `&mut String`，给字符串追加感叹号。

## 常见坑

- `&mut` 同一时间只能有一个，这是数据竞争防线。
- `Copy` 类型赋值后旧变量仍可用，例如整数和布尔值。
- 借用不是复制，借用结束后原所有者仍负责释放资源。
