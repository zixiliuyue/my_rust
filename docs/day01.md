# Day 1：Cargo、变量和基础类型

## 先运行

```bash
cargo run --bin day01_basics
```

你会看到课程名称、基础类型的打印结果，以及栈和只读数据区的说明。

## 再读代码

重点阅读 `src/bin/day01_basics.rs`：

- `fn main()` 是程序入口。
- `let` 默认不可变，`let mut` 才能修改。
- `u8`、`i32`、`f64`、`bool`、`char` 是常见基础类型。
- `println!` 是宏，名字后面的 `!` 表示它不是普通函数。

## 底层原理

Rust 编译流程可以简单理解为：源码经过 `rustc` 编译成机器码，`cargo run` 会先构建再运行。固定大小的基础类型通常放在栈上，函数调用时创建栈帧，函数返回时栈帧整体释放。

`&str` 字面量比较特殊。文本内容被编译进程序的只读区域，变量保存的是指向文本的地址和长度，所以它不是把整段字符串复制到栈上。

## 常见坑

- Rust 变量默认不可变，这是设计选择，不是限制。
- `char` 是 Unicode 标量值，不等于一个字节。
- `println!("{value}")` 里的变量名必须在当前作用域中存在。
